# AI Foundations (AI 900 & Vendor-Neutral Mapping to Azure)

This repository teaches core AI concepts from the ground up for software engineers who are new to AI. It focuses on clear fundamentals first, then maps those concepts to Azure services and tooling where relevant.

This repository will help you pass the AI-900 exam, but the primary goal is to build strong AI foundations and pass the exam as a natural outcome of deep understanding.

## Who this is for
- Junior software engineers who have not worked with AI before
- Developers who want a practical mental model of common AI workloads and the ML lifecycle
- Readers who want to understand both the “why” and the “how” of foundational AI topics

## What you will learn
- What AI is, what ML is, and how common AI workloads differ
- How data becomes model inputs (features) and outputs (labels)
- Why training, validation, and testing exist, and how splitting works
- How to choose the right task type (Classification, Regression, Clustering, Anomaly Detection)
- How evaluation metrics work at a conceptual level (what they mean and when to use them)
- Responsible AI basics, including Transparency and Generative AI Safety Layers
- How language, vision, and document processing workloads fit into real solutions
- How these concepts map to Azure services (reference page)

## How to use this repo
1. Start with the docs in order. Each page builds on the previous one.
2. Read for understanding first, then revisit pages when you need a decision rule or a reminder.
3. Use the Azure mapping page when you want to connect the concept to specific Azure services.

## Practice Tests
This repo includes practice tests to help validate understanding and catch common confusion points. The goal is not only to test recall, but to make sure the content is understandable and usable in real scenarios.

Practice tests come in several formats:
- Markdown files (with answers)
- Microsoft Forms quizzes

Practice tests come in three types:

### Raw Practice Tests (Questions & Correct Answers Only)
This will include all questions with no explanations — just the question and the correct answer.

It acts like the source material.

Markdown will be the primary format for Raw, since it is designed to be a high-volume source set.

- [Practice Test 01 (Raw)](practice-tests/raw/practice-test-01.md)
- [Practice Test 02 (Raw)](practice-tests/raw/practice-test-02.md)

### Detailed Practice Tests (Deep Explanations & References)
This version includes deep explanations, breakdowns, and additional clarifications.

It contains more questions because each concept is expanded.

Detailed practice tests will be available as both:
- Markdown files (for detailed reference)
- Microsoft Forms quizzes (for exam-style practice)

- [Practice Test 01 (Detailed)](practice-tests/detailed/practice-test-01.md)
- [Practice Test 02 (Detailed)](practice-tests/detailed/practice-test-02.md)
- Microsoft Forms: Practice Test 01 (Detailed) (Link TBD)
- Microsoft Forms: Practice Test 02 (Detailed) (Link TBD)

### Compressed Practice Tests (High-Coverage & Low-Volume Review)
This compresses many detailed questions into a smaller number of high-coverage questions.

The goal is to reduce volume while keeping maximum understanding.

Compressed practice tests will be available as both:
- Markdown files (as a compact reference set)
- Microsoft Forms quizzes (for fast, high-impact revision)
